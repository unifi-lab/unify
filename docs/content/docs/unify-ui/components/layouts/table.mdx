# Table Layout

The table layout provides a traditional spreadsheet-like view for displaying structured data. It's ideal for showing multiple records with clear column headers and easy comparison between entries.

```tsx
import { useState } from 'react';
import { UniRender } from '@unilab/unify-ui';

function EditableTableExample() {
  const [data, setData] = useState([
    { id: 1, name: 'John Doe', email: 'john@example.com', role: 'Admin', isActive: true },
    { id: 2, name: 'Jane Smith', email: 'jane@example.com', role: 'User', isActive: true },
    { id: 3, name: 'Bob Johnson', email: 'bob@example.com', role: 'User', isActive: false }
  ]);

  const handleEdit = async (updatedRecord, index) => {
    const newData = [...data];
    newData[index] = updatedRecord;
    setData(newData);
    // Save to backend
    console.log('Updated record:', updatedRecord);
  };

  const handleDelete = async (record, index) => {
    const newData = data.filter((_, i) => i !== index);
    setData(newData);
    // Delete from backend
    console.log('Deleted record:', record);
  };

  return (
    <UniRender
      entity={{
        name: 'User',
        fields: [
          { name: 'id', type: 'number', required: true },
          { name: 'name', type: 'string', required: true },
          { name: 'email', type: 'string', required: true },
          { name: 'role', type: 'string' },
          { name: 'isActive', type: 'boolean' }
        ]
      }}
      data={data}
      layout="table"
      config={{
        id: { label: 'ID', width: '60px', editable: false },
        name: { 
          label: 'Full Name', 
          editable: true,
          required: true,
          type: 'text'
        },
        email: { 
          label: 'Email Address', 
          editable: true,
          required: true,
          type: 'email'
        },
        role: { 
          label: 'Role',
          editable: true,
          type: 'select',
          options: ['Admin', 'User', 'Manager']
        },
        isActive: { 
          label: 'Status',
          editable: true,
          type: 'checkbox'
        }
      }}
      generalConfig={{
        editable: true,
        showActions: true,
        actions: {
          edit: true,
          delete: true
        }
      }}
      onEdit={handleEdit}
      onDelete={handleDelete}
    />
  );
}
```

### Column Configuration

```tsx
const columnConfig = {

  id: {
    label: 'ID',
    width: '60px',
    align: 'center',
    editable: false

    label: 'Full Name',
    editable: true,

    type: 'text'
  },

    label: 'Email Address',
    editable: true,
    required: true,
    type: 'email'
  },
  
  // Select dropdown
  role: {
    label: 'Role',
    editable: true,
    type: 'select',
    options: ['Admin', 'User', 'Manager', 'Guest']
  },
  
  // Number input
  salary: {
    label: 'Salary',
    editable: true,
    type: 'number',
    align: 'right'
  },
  
  // Date picker
  joinDate: {
    label: 'Join Date',
    editable: true,
    type: 'date'
  },
  
  // Checkbox
  isActive: {
    label: 'Active',
    editable: true,
    type: 'checkbox'
  },
  
  // Textarea for long text
  bio: {
    label: 'Biography',
    editable: true,
```

## üîß Advanced Features

### Responsive Tables

Tables automatically adapt to smaller screens by:
- Horizontal scrolling on mobile devices
- Sticky column headers
- Responsive column widths

### Performance Optimization

For large datasets, consider:
- Implementing pagination
- Using virtual scrolling
- Server-side filtering and sorting

```tsx
// Example with pagination
<UniRender
  entity={userEntity}
  data={currentPageData}

  generalConfig={{
    showActions: true,
    pagination: {
      page: currentPage,
      limit: 25,
      total: totalRecords,
      onPageChange: handlePageChange
    }
  }}
/>
```

## üì± Mobile Considerations

On mobile devices, tables can become difficult to read. Consider:
- Using card layout for mobile views
- Implementing responsive breakpoints
- Showing only essential columns on small screens

## üåü Best Practices

1. **Keep it Simple**: Start with basic columns and add complexity as needed
2. **Clear Headers**: Use descriptive column labels
3. **Consistent Formatting**: Align similar data types consistently
4. **Performance**: Implement pagination for large datasets
5. **Accessibility**: Ensure keyboard navigation and screen reader support
