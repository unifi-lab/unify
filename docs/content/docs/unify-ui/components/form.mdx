# Form

The form layout displays data in a detailed form format, showing one record at a time with easy navigation between records. Perfect for detailed viewing and editing of individual items.

## üéõÔ∏è Form Configuration

### Field Layout

```tsx
const formConfig = {
  // Text fields
  firstName: {
    label: 'First Name',
    required: true,
    order: 1
  },
  
  lastName: {
    label: 'Last Name',
    required: true,
    order: 2
  },
  
  // Email with validation
  email: {
    label: 'Email Address',
    type: 'email',
    required: true,
    order: 3
  },
  
  // Phone number
  phone: {
    label: 'Phone Number',
    type: 'text',
    order: 4
  },
  
  // Select dropdown
  department: {
    label: 'Department',
    type: 'select',
    options: ['Engineering', 'Marketing', 'Sales', 'HR'],
    order: 5
  },
  
  // Date picker
  joinDate: {
    label: 'Join Date',
    type: 'date',
    order: 6
  },
  
  // Checkbox
  isManager: {
    label: 'Is Manager',
    type: 'checkbox',
    order: 7
  },
  
  // Textarea for longer text
  bio: {
    label: 'Biography',
    type: 'textarea',
    order: 8
  },
  
  // Hidden fields
  id: {
    hidden: true
  }
};
```

## üß≠ Navigation Controls

The form layout includes built-in navigation controls:

- **Previous/Next buttons**: Navigate between records
- **Record counter**: Shows current position (e.g., "2 of 10")
- **Direct navigation**: Jump to specific record by number
- **Keyboard shortcuts**: Arrow keys for navigation

### Navigation Features

```tsx
// Navigation is automatically included
// No additional configuration needed
<UniRender
  layout="form"
  data={records}
  // ... other props
/>
```

## ‚úèÔ∏è Editing in Form Layout

Form layout is ideal for detailed editing:

```tsx
const handleEdit = (updatedRecord, index) => {
  // Update the record at the specific index
  const newData = [...data];
  newData[index] = updatedRecord;
  setData(newData);
};

<UniRender
  layout="form"
  data={data}
  generalConfig={{
    editable: true,
    showActions: true
  }}
  onEdit={handleEdit}
/>
```

## üîß Advanced Features

### Conditional Fields

Show/hide fields based on other field values:

```tsx
const conditionalConfig = {
  isManager: {
    label: 'Is Manager',
    type: 'checkbox'
  },
  
  managementLevel: {
    label: 'Management Level',
    type: 'select',
    options: ['Team Lead', 'Department Head', 'VP', 'C-Level'],
    // This field would be conditionally shown
    // when isManager is true
  }
};
```

### Validation

Built-in validation for different field types:

```tsx
const validatedConfig = {
  email: {
    label: 'Email Address',
    type: 'email',
    required: true,
    // Email format validation automatic
  },
  
  phone: {
    label: 'Phone Number',
    type: 'text',
    pattern: '^\\+?[1-9]\\d{1,14}$',
    // Custom validation pattern
  },
  
  salary: {
    label: 'Salary',
    type: 'number',
    min: 0,
    max: 1000000,
    // Numeric range validation
  }
};
```
